Description

Before starting this project, it is necessary to determine the requirements for the web application and design. First thing, service users must be able to do the following things via API access:

  • Create a new game of darts on the server. The game can be closed (invitation-only) or open to everyone (any player can connect, including the creator);
  • Get the information about current games available for connection;
  • Connect to open or closed games;
  • Get the current status of the game and statistics;
  • Transmit the data about your throws.

The logic of the service must carry out the following (internal logic of the service):

  • Authentication and authorization of users;
  • Game management: creation, status definition, completion;
  • Database maintenance;
  • Validation of transferred player data as per the game rules;
  • Scoring, ordering, setting of transition speed, result output;
  • In-game statistics management: average number of points per turn, number of moves per game, game leaders;
  • Auxilliary functions: calculation of possible endings, and more.
Objectives

In this stage, our goal is to run a Spring Boot web application and create several endpoints.

  • Create and run a Spring Boot application on the 28852 port.
  • Implement the following API structure for the game logic:
  1. POST api/game/create to create a game;
  2. GET api/game/list to get the list of current games on the server;
  3. GET api/game/join to join to a game;
  4. GET api/game/status to get the game status;
  5. POST api/game/throws to send the information about the player's throws.

All API's must respond with the status 200 OK code and the next JSON:

{
   "status": "Under construction!"
}
